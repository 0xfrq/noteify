\section{Mengenal Non-Deterministic Finite Automata (NFA)}

\subsection{Kenapa Harus Ada NFA?}
Di materi sebelumnya, kita udah bahas DFA yang sifatnya kaku dan serba pasti. Kalau di DFA, tiap \textit{state} wajib punya satu jalan keluar untuk setiap simbol, nggak boleh lebih dan nggak boleh kurang.

Tapi pada praktiknya, pas kita mau mendeteksi pola bahasa yang agak rumit, ngerancang DFA itu bisa bikin pusing karena \textit{state}-nya jadi meledak (banyak banget). Nah, di sinilah \textit{Non-Deterministic Finite Automaton} (NFA) jadi pahlawan. NFA ini ngasih kita kebebasan penuh buat ngerancang mesin dengan cara yang lebih fleksibel dan sesuai sama intuisi manusia.

Di NFA, kalau mesin dapet satu input, dia bisa bereaksi dengan tiga cara ini:
\begin{itemize}
    \item \textbf{Multiple Paths (Bercabang):} Satu karakter bisa bikin mesin loncat ke lebih dari satu \textit{state} sekaligus. Mesin bakal seolah-olah "ngekloning" dirinya sendiri buat nyusurin semua jalan itu.
    \item \textbf{Dead End (Jalan Buntu):} Kalau mesin dapet karakter yang nggak ada panah transisinya, kloningan mesin di jalur itu langsung mati atau gagal.
    \item \textbf{Epsilon Transition ($\epsilon$):} Ini kemampuan buat loncat atau pindah ke \textit{state} lain secara gaib, tanpa perlu ngebaca karakter apa pun dari pita input!
\end{itemize}

\subsection{Aturan Main Diterimanya String di NFA}
Karena mesin NFA bisa bercabang jadi banyak kloningan pas lagi ngebaca string, syarat biar string itu \textbf{diterima} (\textit{Accept}) simpel banget:
\textbf{Asalkan ada minimal SATU aja jalur kloningan yang sukses berhenti di Final State pas string-nya habis dibaca, maka string itu dianggap sah dan diterima.}

Nggak peduli kalau ada puluhan kloningan lain yang mati di jalan buntu atau nyasar ke \textit{state} biasa. Kalau semuanya gagal, baru deh string itu dinyatakan \textbf{ditolak} (\textit{Reject}).

\section{Kumpulan Studi Kasus Perancangan NFA}

Biar makin paham bedanya NFA dan DFA, mari kita bedah berbagai macam studi kasus. Merancang NFA itu rasanya jauh lebih gampang, karena kita tinggal bikin jalur khusus buat pola yang bener aja, sisanya biarin buntu!

\subsection{Kasus 1: Pola Berakhiran "01"}
Kita mau bikin mesin yang nerima string biner, asalkan dua huruf terakhirnya adalah "01". Depannya mau angka apa aja, sepanjang apa pun, bebas. Alfabetnya $\Sigma=\{0,1\}$.

\begin{figure}[h]
    \centering
    \begin{tikzpicture}[shorten >=1pt,node distance=2.5cm,on grid,auto, thick]
       \node[state,initial,initial text=Start] (q0) {$q_0$};
       \node[state] (q1) [right=of q0] {$q_1$};
       \node[state,accepting] (q2) [right=of q1] {$q_2$};

       \path[->] 
       (q0) edge [loop above] node {0, 1} (q0)
            edge node {0} (q1)
       (q1) edge node {1} (q2);
    \end{tikzpicture}
    \caption{NFA String Berakhiran "01"}
\end{figure}

\textbf{Kenapa NFA ini gampang banget dibaca?}
\begin{itemize}
    \item Di state $q_0$, kalau dia dapet angka '0', mesin ini **nebak** atau **bercabang**. Satu kloningan muter tetep di $q_0$ (berasumsi ini bukan '0' yang terakhir), dan satu kloningan lagi maju ke $q_1$ (berasumsi ini adalah awal dari akhiran "01").
    \item Kalau asumsinya salah (misal abis '0' malah '0' lagi), kloningan yang ada di $q_1$ bakal **mati** karena nggak ada panah '0' dari $q_1$. Tapi kloningan yang di $q_0$ masih hidup dan siap nebak lagi.
\end{itemize}

\subsection{Kasus 2: Mengandung Substring "010"}
Sekarang kita cari string yang di tengah-tengahnya (atau di mana pun) punya pola berurutan "010". Kalau udah ketemu "010", belakangnya mau diisi apa aja bebas.

\begin{figure}[h]
    \centering
    \begin{tikzpicture}[shorten >=1pt,node distance=2.5cm,on grid,auto, thick]
       \node[state,initial,initial text=Start] (q0) {$q_0$};
       \node[state] (q1) [right=of q0] {$q_1$};
       \node[state] (q2) [right=of q1] {$q_2$};
       \node[state,accepting] (q3) [right=of q2] {$q_3$};

       \path[->] 
       (q0) edge [loop above] node {0, 1} (q0)
            edge node {0} (q1)
       (q1) edge node {1} (q2)
       (q2) edge node {0} (q3)
       (q3) edge [loop above] node {0, 1} (q3);
    \end{tikzpicture}
    \caption{NFA Mendeteksi Substring "010"}
\end{figure}

\textbf{Alur Logika:}
State $q_0$ tugasnya cuma nunggu. Tiap kali ada angka '0' lewat, dia ngutus satu kloningan buat masuk ke terowongan $q_1 \rightarrow q_2 \rightarrow q_3$. Kalau di tengah jalan urutannya rusak (misal abis '0' malah '0' lagi di $q_1$), kloningan itu bakal gugur. Tapi kalau berhasil lolos sampai $q_3$, dia bakal santai nge-loop di sana sampai string habis.

\subsection{Kasus 3: Karakter Ketiga dari Belakang adalah "1"}
Ini adalah contoh klasik di mana NFA jauh menang telak dibanding DFA. Kita mau nerima string yang kalau dihitung dari kanan ke kiri, karakter ketiganya pasti angka '1'.
Contoh yang diterima: "00\textbf{1}00", "\textbf{1}11", "0101\textbf{1}01".

\begin{figure}[h]
    \centering
    \begin{tikzpicture}[shorten >=1pt,node distance=2.5cm,on grid,auto, thick]
       \node[state,initial,initial text=] (q0) {$q_0$};
       \node[state] (q1) [right=of q0] {$q_1$};
       \node[state] (q2) [right=of q1] {$q_2$};
       \node[state,accepting] (q3) [right=of q2] {$q_3$};

       \path[->] 
       (q0) edge [loop above] node {0, 1} (q0)
            edge node {1} (q1)
       (q1) edge node {0, 1} (q2)
       (q2) edge node {0, 1} (q3);
    \end{tikzpicture}
    \caption{NFA: Karakter ketiga dari belakang adalah 1}
\end{figure}

\textbf{Rahasia Kekuatan NFA:}
Bayangin kalau kita bikin ini pakai DFA, kita butuh \textit{state} buat nginget semua kombinasi 3 huruf terakhir (kayak 000, 001, 010, dst), total butuh 8 \textit{state} yang ribet panahnya.
Tapi pakai NFA, kita cuma butuh 4 \textit{state}! Tiap ketemu angka '1', $q_0$ ngutus kloningan maju ke $q_1$. Kloningan ini dikasih jatah persis dua langkah lagi ke $q_3$. Kalau stringnya keburu habis sebelum nyampe $q_3$, atau stringnya masih sisa pas dia udah di $q_3$ (mati, karena $q_3$ nggak punya panah keluar), berarti itu bukan '1' yang ketiga dari belakang. Cerdas banget, kan?

\subsection{Kasus 4: Menggunakan Jalan Tol Epsilon ($\epsilon$)}
Kita mau deteksi string yang punya "11" berurutan \textbf{ATAU} "00" berurutan. Ini gampang banget diselesaikan pakai \textit{Epsilon Transition} (loncat gaib tanpa baca karakter).

\begin{figure}[h]
    \centering
    \begin{tikzpicture}[shorten >=1pt,node distance=2.5cm,on grid,auto, thick]
       \node[state,initial,initial text=Start] (S) {$S$};
       
       % Cabang atas untuk 11
       \node[state] (q1) [above right=1.5cm and 2cm of S] {$q_1$};
       \node[state] (q2) [right=of q1] {$q_2$};
       \node[state,accepting] (q3) [right=of q2] {$q_3$};

       % Cabang bawah untuk 00
       \node[state] (p1) [below right=1.5cm and 2cm of S] {$p_1$};
       \node[state] (p2) [right=of p1] {$p_2$};
       \node[state,accepting] (p3) [right=of p2] {$p_3$};

       \path[->] 
       (S) edge node [above left] {$\epsilon$} (q1)
           edge node [below left] {$\epsilon$} (p1)
           
       (q1) edge [loop above] node {0, 1} (q1)
            edge node {1} (q2)
       (q2) edge node {1} (q3)
       (q3) edge [loop above] node {0, 1} (q3)
       
       (p1) edge [loop below] node {0, 1} (p1)
            edge node {0} (p2)
       (p2) edge node {0} (p3)
       (p3) edge [loop below] node {0, 1} (p3);
    \end{tikzpicture}
    \caption{NFA Epsilon untuk mendeteksi substring "11" atau "00"}
\end{figure}

Pas mesin dinyalain, berkat transisi $\epsilon$, mesin seketika itu juga terbelah jadi dua tanpa perlu baca input apa pun. Satu kloningan \textit{standby} di $q_1$ tugasnya nyari "11". Kloningan satunya lagi \textit{standby} di $p_1$ tugasnya nyari "00". Mereka kerja bareng secara paralel. Siapa yang berhasil nyampe garis \textit{finish} duluan ($q_3$ atau $p_3$), dialah yang bikin stringnya diterima.

\subsection{Kasus 5: Pengecekan Pola Terstruktur $(a \cup b)^* a b b$}
Kita mau konversi ekspresi reguler $(a \cup b)^* a b b$ ke dalam NFA. Artinya, depannya boleh diisi kombinasi huruf 'a' atau 'b' sebanyak apa pun, asalkan ditutup dengan tepat pola "abb". Alfabetnya $\Sigma=\{a,b\}$.

\begin{figure}[h]
    \centering
    \begin{tikzpicture}[shorten >=1pt,node distance=2.5cm,on grid,auto, thick]
       \node[state,initial,initial text=] (q0) {$q_0$};
       \node[state] (q1) [right=of q0] {$q_1$};
       \node[state] (q2) [right=of q1] {$q_2$};
       \node[state,accepting] (q3) [right=of q2] {$q_3$};

       \path[->] 
       (q0) edge [loop above] node {a, b} (q0)
            edge node {a} (q1)
       (q1) edge node {b} (q2)
       (q2) edge node {b} (q3);
    \end{tikzpicture}
    \caption{NFA untuk pola regex $(a \cup b)^* a b b$}
\end{figure}

Simulasi kalau kita kasih input "ababb":
\begin{itemize}
    \item Baca 'a' ke-1: Mesin bercabang. Ada yang stay di $q_0$, ada yang maju ke $q_1$.
    \item Baca 'b' ke-2: Yang di $q_0$ tetep di $q_0$. Yang di $q_1$ maju ke $q_2$.
    \item Baca 'a' ke-3: Yang di $q_0$ bercabang lagi ke $q_0$ dan $q_1$. Yang di $q_2$ mati (karena dari $q_2$ butuhnya 'b').
    \item Baca 'b' ke-4: Kloningan dari $q_0$ tetep di $q_0$. Kloningan dari $q_1$ maju ke $q_2$.
    \item Baca 'b' ke-5: Kloningan dari $q_0$ tetep di $q_0$. Kloningan dari $q_2$ maju ke $q_3$.
    \item String habis! Karena ada satu kloningan yang sukses nangkring di $q_3$ (\textit{Final State}), string "ababb" \textbf{Diterima}.
\end{itemize}

\subsection{Kasus 6: Bahasa Tanpa Salah Satu Huruf (Missing Letter)}
Misalkan kita punya alfabet dengan 3 huruf $\Sigma=\{a, b, c\}$. Kita mau mesin nerima string asalkan ada \textbf{salah satu huruf yang nggak pernah muncul} sama sekali.
Pilihannya: String nggak punya 'a', ATAU nggak punya 'b', ATAU nggak punya 'c'.

\begin{figure}[h]
    \centering
    \begin{tikzpicture}[shorten >=1pt,node distance=3cm,on grid,auto, thick]
       \node[state,initial,initial text=Start] (S) {$S$};
       
       \node[state,accepting] (qA) [above right=1.5cm and 2.5cm of S] {$q_A$};
       \node[state,accepting] (qB) [right=3cm of S] {$q_B$};
       \node[state,accepting] (qC) [below right=1.5cm and 2.5cm of S] {$q_C$};

       \path[->] 
       (S) edge node [above left] {$\epsilon$} (qA)
           edge node [above] {$\epsilon$} (qB)
           edge node [below left] {$\epsilon$} (qC)
           
       (qA) edge [loop right] node {b, c} (qA)
       (qB) edge [loop right] node {a, c} (qB)
       (qC) edge [loop right] node {a, b} (qC);
    \end{tikzpicture}
    \caption{NFA String Kehilangan Minimal Satu Huruf Alfabet}
\end{figure}

\textbf{Alur Logikanya:}
Begitu jalan, $\epsilon$-transition bikin mesin pecah jadi 3 divisi pemeriksa:
1. Divisi $q_A$ tugasnya nampung string yang \textbf{nggak ada 'a'}. Makanya dia cuma mau nerima huruf 'b' dan 'c'. Kalau ada 'a' nyasar masuk, divisi ini langsung gagal/mati.
2. Divisi $q_B$ khusus nampung string yang \textbf{nggak ada 'b'} (cuma nerima 'a' dan 'c').
3. Divisi $q_C$ khusus nampung string yang \textbf{nggak ada 'c'} (cuma nerima 'a' dan 'b').

Kalau kita kasih input "bcbcbc", divisi $q_B$ dan $q_C$ bakal rontok di tengah jalan. Tapi divisi $q_A$ bakal bertahan muter-muter terus dan sukses. Karena minimal ada satu divisi yang \textit{survive} sampai akhir, string "bcbcbc" dinyatakan sah dan \textbf{diterima}!
